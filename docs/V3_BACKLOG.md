# V3 需求候选与技术待办

状态：`PLANNED`。本文只登记 V3 候选需求，不代表已经确定 V3 的完整范围、版本号或发布时间。

## 1. 受管最大化按钮图标专项

### 背景

V2 内置窗口组在受管最大化时，Chrome 右上角中间按钮的点击行为正常：再次点击可以还原窗口组，最小化、还原、F11、Snap、任务栏恢复及异常恢复也已通过。但是 Chrome 按钮仍可能显示“最大化”图标，而不是传统的“还原”图标。这是非关键视觉差异，不影响 V2 的窗口可达性和恢复能力，因此按用户决定从 V2 发布门禁移至 V3。

### V3 研究要求

- 先确认图标由 Chrome 非客户区状态、窗口 placement、DWM 或 Chrome 自绘状态中的哪一层决定；
- 优先使用公开、可回滚的 Win32/DWM 能力，不得仅为改变图标而破坏真实窗口状态；
- 不得注入 Chrome/Explorer，不得使用 `SetParent`，不得安装全局 Hook 或驱动；
- 不得使用全局 AppBar 占用工作区，也不得采用不能精确恢复的窗口样式修改；
- 若无法在保持 V2 安全不变量的前提下修正，保留现状并记录技术限制，不以视觉一致性换取窗口不可达风险。

### V3 验收标准

自动验收至少覆盖：最大化、还原、最小化、F11、左右 Snap、异常终止恢复、Explorer 重建和原始窗口样式精确恢复。

人工验收必须确认：按钮图标与行为一致；Chrome 原生网页标签、最小化/还原/关闭按钮仍可用；任务栏单入口、组移动和安全退出无回归。若未能安全修正，用户明确接受记录的视觉限制即可关闭该 V3 研究项。
