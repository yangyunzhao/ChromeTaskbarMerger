# ChromeTaskbarMerger V4 功能需求池

> 状态：`BACKLOG`
>
> 建立日期：2026-07-17

V4 用于登记会扩大产品能力的新功能需求。登记不代表已经批准开发、确定版本号或承诺发布时间。
V3 原则上只处理易用性、美观性和缺陷修复；开发 V3 时出现的功能建议先放在本文档，待 V3 完成后
统一评估价值、风险、兼容性和 Phase 划分。

## 已登记候选

1. 多个可命名窗口组和持久工作区；
2. 标签拖放合组、拆组和跨组移动；
3. 任务栏代理窗口、每标签 DWM 缩略图和当前标签图标；
4. 完整主题系统、更多标签尺寸以及可配置快捷键；
5. 更丰富的 Chrome profile 显示信息和 favicon；
6. 支持 Edge、Brave 等其他 Chromium 浏览器；
7. 面向虚拟桌面和多显示器的独立分组策略；
8. 任意自由拖放标签栏或保存多套标签栏布局。

## V4 启动门禁

V4 正式规划前至少需要：

- V3 已完成或明确停止；
- 对候选需求去重并确定优先级；
- 明确哪些能力属于同一版本，避免把互不相关的大功能塞入一个 Phase；
- 新建独立 `V4_REQUIREMENTS.md` 和测试计划；
- 重新评估恢复日志、隐私、Chrome 兼容性和便携发布边界。
